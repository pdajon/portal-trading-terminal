## 1. How To Use This Pack

Use this as the reusable prompt layer for Portal. The condensed Brand Bible defines the rules; this pack turns those rules into copy/paste prompts for design, image generation, and implementation.

- Paste the Global Portal Context before any major UI request.
- Use the graphic prompts for landing visuals, social posts, product mockups, AI orb visuals, and brand assets.
- Use the Codex prompts when asking an agent to change the actual app UI.
- Keep prompts direct. Do not let Codex or image tools freestyle a new brand every time.

### Global Portal Context Prompt

```text
Design this for Portal, a premium crypto trading terminal built around custom execution, AI coaching, discipline rules, and chart-based automation.

Portal's design language is a balanced hybrid: luxury trading cockpit as the app foundation, cyber wormhole energy as the brand layer.

The UI should feel dark, premium, technical, calm, and powerful. Use deep void black and blue-black surfaces, medium Apple-glass panels, restrained cyan/blue glow, subtle purple/magenta energy, and dimensional depth. Gold/yellow is special emphasis only.

Portal's locked rectangular edge radius is 7px. Apply it to buttons, inputs, panels, cards, popovers, overlays, modals, window shells, and clipped glass surfaces. Keep true circular geometry circular.

The chart is sacred: clean by default, powerful by intention. Advanced data should be reachable through drawers, tabs, expandable panels, overlays, or layer controls - not jammed into the default screen.

Avoid cluttered exchange UI, rainbow neon, generic SaaS dashboards, casino visuals, fake luxury templates, too many pills, too many borders, and card-inside-card clutter.

Everything must pass this test: does it help the trader act, wait, review, or avoid a mistake?
```

## 4. UI Implementation Prompts

### Codex UI Implementation Master Prompt

```text
Implement this UI change inside Portal using the Portal Brand Bible rules.

Do not add more containers, pills, borders, or decorative clutter to solve layout problems. Prefer cleaner hierarchy, better spacing, fewer surfaces, and clearer grouping.

Portal should feel like a luxury trading cockpit with cyber wormhole energy only where earned. The chart remains dominant. The UI should use progressive disclosure: show core information by default and tuck advanced details into expandable areas.

Use dark glass panels, soft borders, restrained cyan/blue active states, purple/magenta AI states, and red/amber for serious warning states. Glow only when active, armed, live, selected, warning, or AI-thinking.

Use the locked Portal rectangular edge radius of 7px for new buttons, inputs, panels, cards, popovers, overlays, modals, window shells, and clipped glass surfaces.

Before editing, identify the current UI clutter problem, the exact component boundaries, and the state logic. Then implement the smallest clean change that improves clarity without breaking layout.
```

### Footer Command Deck Fix Prompt

```text
Redesign the Portal bottom footer as a mode-based command deck.

Top-level footer tabs should switch the entire footer layout. Trade, Discipline, Journal, AI Coach, Session Logs, and Settings should each own the footer when selected.

When Discipline, Journal, AI Coach, Session Logs, or Settings is selected, trade buttons should not remain visible.

Use the full footer footprint intelligently. Avoid bunching everything in the center. Reduce containers, pills, borders, and nested boxes. The answer is less clutter, not more decoration.

Trade mode should show a clean execution layout: size input, long/short controls, close/manage controls when in position, available margin, key risk info, and expandable advanced options.

The footer should feel like a command surface, not a storage drawer.
```

### Discipline Rules UI Prompt

```text
Design the Portal Discipline Rules surface as a trader safety system, not a settings page.

Rules should feel like contracts the trader makes with themselves. Group rules by purpose: loss limits, cooldowns, position sizing, session goals, no-trade zones, R:R requirements, and checklist requirements.

Each rule card should show rule name, trigger condition, consequence, current state, and edit/disable/override controls.

States: inactive, active, armed, triggered, locked, overridden.

Warnings should be firm and blunt, visually serious but not panic-inducing. Override flows should feel deliberate and be logged.
```

### AI Coach UI Prompt

```text
Design the Portal AI Coach as a tactical trading coach with a futuristic system-intelligence visual layer.

The AI should not feel like a generic chatbot. It should appear as a Portal signal/orb, compact insight cards, inline warnings, and deeper review panels when opened.

AI copy should be calm, observant, direct, and useful.

Good examples:
- You entered before confirmation.
- You are sizing up after a loss. That is revenge risk.
- This setup matches your low-win-rate pattern.

Use cyan/purple energy accents, subtle thinking pulse, dimensional edge lighting, and compact insight cards.
```

### Journal UI Prompt

```text
Design the Portal Journal as a hybrid auto-captured trade review system.

The app should auto-capture trade time, symbol, direction, entry, exit, PnL, chart state, rules active, AI warnings, and session context.

The user should only add high-value human context: thesis, emotion, mistake, lesson, confidence, setup quality.

The UI should be calmer and more spacious than the trade panel. This is reflection mode, not execution mode.
```

## 6. Approval Checklist

Before using or approving any Portal output, check it against this list:

- Does it feel like Portal: luxury trading cockpit plus cyber wormhole energy?
- Is the chart protected as the main surface?
- Is the glow earned?
- Is the state readable in under one second?
- Is advanced information reachable without being shoved forward?
- Does the AI feel like a Portal signal, not a support chatbot?
- Does the copy avoid corporate filler and fake hype?
- Could this be simpler?

Final rule: Portal should feel clean, serious, dimensional, and alive - not busy for the sake of looking advanced.
