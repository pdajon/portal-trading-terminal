# PORTAL

Brand Bible - Condensed v1.0
Cosmic precision for a premium trading terminal

Core direction: luxury trading cockpit + cyber wormhole energy

> Tagline benchmark
> Trade Anything. Any Way.  -  CTA benchmark: Enter Portal

## 1. Brand Essence

Portal is a premium crypto trading command center for serious operators. The product should feel like stepping into a clean, intelligent, slightly dangerous wormhole built for decisions under pressure.

> Brand essence
> Cosmic precision: dark cockpit control, dimensional energy, clear execution, and discipline under pressure.

### What Portal Should Feel Like

- Luxury trading cockpit as the app foundation.
- Cyber wormhole energy as the brand layer.
- Calm under pressure, technical, sharp, and operator-grade.
- A market observatory, command center, and execution terminal - not a generic exchange clone.

### What Portal Is Not

- Not a crowded exchange interface.
- Not a rainbow neon spaceship dashboard.
- Not generic SaaS gray.
- Not casino-like, meme-heavy, or fake luxury.
- Not a chatbot slapped onto a chart.

### Visual North Star

The wormhole image defines the physics of the brand: dark center of gravity, light bending around a focal point, glitch-like color separation, motion trails, depth, and controlled chaos around a clean center.

> Design rule
> The UI should borrow the feeling of the wormhole, not paste wormholes everywhere. The trading surface stays clean; brand moments can roar.

## 2. Color, Type, and Surface System

### Color Hierarchy

- Cyan / blue = primary Portal identity: clean, technical, trustworthy.
- Purple / magenta = AI, dimensional energy, futuristic glow.
- Gold / yellow = special emphasis only. Too much gold starts looking like Binance cosplay.

### Core Tokens

| Token | Hex | Use |
| --- | --- | --- |
| Void Black | #05060A | Deepest app background |
| Deep Space | #090B14 | Shell / large surfaces |
| Panel Black | #0D101B | Panels and cards |
| Glass Navy | #111827 | Glass UI surfaces |
| Portal Cyan | #33F5FF | Primary active accent |
| Electric Blue | #3B82F6 | CTA / selected states |
| Plasma Purple | #8B5CF6 | AI / dimensional energy |
| Signal Pink | #FF3D8A | High-energy brand moments |
| Warning Amber | #F59E0B | Caution / approval needed |
| Profit Green / Loss Red | #22C55E / #EF4444 | PnL and trade direction |

### Typography

- Clean sans-serif for labels, navigation, copy, buttons, and settings.
- Mono treatment for prices, PnL, percentages, timestamps, wallet addresses, order IDs, and logs.
- Headlines should be short, commanding, and never motivational fluff.

### Surface Rules

- Medium dark glass inside the app.
- Heavier glass in landing pages and marketing visuals.
- Soft borders by default; glow only when active, armed, live, selected, warning, or AI-thinking.
- Glass should add depth, never hurt readability.
- Portal's locked rectangular edge radius is 7px. Use it for buttons, inputs, panels, cards, popovers, overlays, modals, window shells, and clipped glass surfaces. Preserve true circles only for avatars, dots, toggles, loading rings, and other circular geometry.

## 3. Layout Philosophy

> The chart is sacred
> Portal is chart-first. The chart should be clean by default and powerful by intention.

### Progressive Disclosure

Portal should avoid the common exchange problem: useful data shoved onto one screen until the trader is visually fighting the interface. Important information should be reachable, not jammed into the default view.

- Show what matters now.
- Tuck advanced data into drawers, tabs, expandable panels, overlays, or detail states.
- Use full available space intelligently; do not bunch everything into the center.
- Group controls by trading intent, not by developer logic.

### Trading Screen Priority

- 1. Chart
- 2. Current position
- 3. Available balance / margin state
- 4. Entry and exit controls
- 5. Active rules
- 6. AI warnings
- 7. Logs and history only when intentionally opened

### Footer Command Deck

The bottom footer should be a mode-based command deck. Major tabs switch the whole footer layout: Trade, Discipline, Journal, AI Coach, Session Logs, Settings.

> Footer rule
> Do not keep trade buttons visible in Discipline, Journal, or Logs. The selected mode owns the footer completely.

### Chart Layers

- Default chart state: price action, active position, essential markers, minimal status.
- Intentional layers: trendline triggers, Fibonacci logic, no-trade zones, AI notes, risk zones, journal markers.
- Every overlay must be hideable, removable, or grouped into a layer control.
- Glow only when a layer is armed, active, near-trigger, triggered, or violated.

## 4. Component Rules

### Buttons

- Normal controls: minimal, clean, terminal-like.
- Rectangular buttons use Portal's locked 7px radius.
- Primary actions: restrained cyan/blue energy treatment.
- Trading actions: compact, fast to read, clear long/short/close separation.
- Danger actions: red/amber, visually separated, confirmation when serious.
- Secondary actions: quiet border, muted text, no glow.

> Button rule
> Only the important shit should look important.

### Cards

Every card needs a job: show status, ask for a decision, explain risk, display performance, control a setting, or reveal insight. If a card has no job, delete it.

### Tabs

- Major tabs should feel like switching operating modes.
- Inactive tabs stay quiet.
- Active tab gets cyan/blue edge, subtle glow, or gradient underline.
- Avoid bulky pill overload.

### Inputs

- Inputs should feel like command fields.
- Trade inputs use mono values and immediate validation.
- Text inputs for journal/AI can breathe more.
- Error states must explain what is wrong and how to fix it.

### Glow Scale

- Level 0: no glow - default UI.
- Level 1: edge glow - selected tab, active input.
- Level 2: aura glow - armed trade, active rule, AI thinking.
- Level 3: pulse glow - near trigger, live intervention.
- Level 4: emergency glow - liquidation risk or major violation.

> Glow rule
> Most of Portal should live at Level 0-1. If everything glows, nothing matters.

## 5. AI, Discipline, and Voice

### AI Assistant

The AI should be a tactical trading coach with a futuristic system-intelligence visual layer. It should feel like a Portal signal/orb, not a generic support chatbot.

- Core presence: floating command orb / Portal signal.
- Visual language: cyan/purple edge, subtle pulse, dimensional lighting.
- States: idle, watching, thinking, insight available, warning, critical intervention.
- Use compact insight cards first; expand for detail only when needed.

### AI Copy Examples

- “You entered before confirmation.”
- “You are sizing up after a loss. That is revenge risk.”
- “This setup matches your low-win-rate pattern.”
- “Wait for candle close.”

### Discipline System

Discipline features should feel like trader safety contracts, not settings buried in a form. They protect the trader from pressure, impulse, and rule-breaking.

- Rule groups: loss limits, cooldowns, position sizing, session goals, no-trade zones, R:R requirements, checklist requirements.
- Rule states: inactive, active, armed, triggered, locked, overridden.
- Warnings should be firm and blunt, visually serious but not panic-inducing.
- Overrides should feel deliberate and be logged.

### Voice Modes

- System Mode: calm, precise, neutral. “Order submitted.”
- Coach Mode: tactical, observant, direct. “You entered before confirmation.”
- Discipline Mode: firm, blunt, protective. “Trade blocked. Cooldown is still active.”

> Voice rule
> No fake hype. No corporate filler. High-pressure moments use fewer words, not more.

## 6. Landing Page, Socials, and Brand Moments

### Landing Page Direction

The landing page should hit with cinematic Portal energy first, then immediately make the product clear. The first impression should feel powerful, but users should know what Portal does without scrolling forever.

> Current hero structure
> Trade Anything. Any Way.
> A Hyperliquid-powered trading terminal for custom execution, AI coaching, discipline rules, and chart-based automation.
> CTA: Enter Portal

### Hero Rules

- Use wormhole visuals more intensely than the app UI.
- Keep text near the dark gravitational focal point.
- Do not place important copy over noisy light trails.
- Use cyan/blue and purple/magenta as the main energy system.
- Keep “Powered by Hyperliquid” visible but quiet.

### Social Content Modes

- Premium Mode: trust, polish, product credibility.
- Cyber Signal Mode: high-energy launch posts, feature teasers, AI/discipline alerts, scroll-stoppers.
- Operator Mode: trader lessons, behavior insights, chart breakdowns, discipline content.

### Marketing Imagery

- Use wormholes, black holes, light trails, glitch energy, futuristic cockpits, dark glass interfaces, and terminal overlays.
- Avoid generic coins, rockets, Wall Street stock photos, and influencer-guru clichés.
- Product mockups should feel clean, dark, glassy, chart-first, and powerful but not cluttered.

### Sound Direction

Portal can eventually develop a cyber/cinematic sound identity: futuristic, clean, not arcade-like, not casino-like. In-app sounds stay controllable and restrained; brand moments can go bigger.

## 7. Reusable Prompt and Approval Checklist

### Global UI Prompt

Design this for Portal, a premium crypto trading terminal built around custom execution, AI coaching, discipline rules, and chart-based automation. The design language is a balanced hybrid: luxury trading cockpit as the base, cyber wormhole energy as the brand layer. Keep the chart dominant. Use dark glass, cyan/blue accents, purple/magenta AI energy, progressive disclosure, and restrained glow only when earned. Avoid cluttered exchange UI, generic SaaS dashboards, rainbow neon, and random decorative pills.

### Feature Prompt Pattern

- State the feature and the mode it belongs to: Trade, Discipline, Journal, AI Coach, Logs, Settings.
- Say what the user is trying to accomplish.
- Define which information must be visible by default.
- Define what should be tucked away.
- Call out the required active/warning/error states.
- End with the Portal approval checklist.

### Anti-Patterns

- Too many cards inside cards.
- Too many pills, borders, and containers.
- Every button glowing.
- Logs visible by default in the trade panel.
- Secondary data fighting the chart.
- AI opening huge chatbot walls during execution.
- Cute copy during serious trading states.
- Over-explaining during trade execution.

### Component Approval Checklist

- What job does this component do?
- Is the state obvious in under one second?
- Is glow earned?
- Does it protect the chart as the main surface?
- Is advanced info reachable without being shoved forward?
- Does this feel like Portal or generic fintech?
- Could this be simpler?
- Does this help the trader act, wait, review, or avoid a mistake?

> Final brand promise
> Portal helps traders trade their way, execute with control, and stop doing dumb shit when pressure hits.
